# Plans

This folder contains pre-planning documents, idea explorations, and roadmaps for the project. It is the upstream thinking that feeds into specs and implementation.

Documents here are often exported conversations from planning sessions (Gemini, Claude, etc.), rough milestone breakdowns, or early-stage feature explorations. They are not expected to be polished — they are working documents.

## What belongs here

- Exported planning conversations and idea explorations
- Phase plans and milestone breakdowns
- Feature roadmaps (what we want to build and roughly when)
- Early-stage explorations that haven't yet become specs
- Prioritization notes and tradeoff discussions

## What does not belong here

- Finalized technical specifications (those go in docs/specs/)
- Architecture decisions (those go in docs/ARDs/)
- Procedures and runbooks (those go in docs/SOPs/)

## Naming convention

No strict convention — use descriptive names with dates where helpful.
Examples: `2026-06-phase-2-planning.md`, `feature-idea-notifications.md`
