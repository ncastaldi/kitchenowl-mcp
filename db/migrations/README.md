# db/migrations

Alembic migration files. Each migration is a versioned, reversible change to the database schema.

## Rules

- Never edit a migration that has already been applied to any environment — create a new one instead
- Migration filenames are auto-generated by Alembic and include a revision hash — do not rename them
- Each migration should do one logical thing (e.g. add a table, add a column, add an index)

## Common commands

```bash
# Create a new migration (auto-detects changes from SQLAlchemy models)
alembic revision --autogenerate -m "short description of change"

# Apply all pending migrations
alembic upgrade head

# Roll back one migration
alembic downgrade -1

# Check current migration state
alembic current

# Show migration history
alembic history
```
